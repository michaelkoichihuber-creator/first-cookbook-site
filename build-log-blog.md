Build Log: Human vs AI

Project



Washoku — Simple Japanese Cooking Blog



The Goal



The purpose of this project was not just to build a website. It was to understand where AI provides leverage and where human judgement is still required.



Where AI Dramatically Sped Me Up

Initial Project Setup



AI helped generate the initial React + Vite project structure much faster than I could have built it manually.



It created:



Component structure

Page structure

Routing

CSS organization

Initial content

Navigation system



Without AI, I would have spent significant time searching documentation and deciding how to organize files. AI gave me a working foundation immediately.



Building Reusable Components



AI quickly generated reusable components such as:



Navbar

RecipeCard

Page layouts



This saved time writing repetitive code and allowed me to focus on reviewing and understanding the implementation instead of building everything from scratch.



SEO Improvements



When implementing SEO, AI rapidly identified several improvements:



Per-page titles

Meta descriptions

Open Graph tags

Semantic HTML improvements

Accessibility fixes



It also generated a reusable useMeta hook rather than duplicating metadata logic across every page.



This would have taken much longer to research and implement manually.



Where Human Judgement Was The Deciding Factor

Choosing The Site Concept



AI did not decide to build a Japanese cooking blog.



I chose the overall direction and theme of the project. The concept needed to feel cohesive and interesting, and that decision came from me.



Deciding What Content Belonged On The Site



AI could generate content, but I had to decide:



Which recipes fit the theme

What tone the site should have

What pages were necessary

What information visitors should see first



These decisions required judgement about the user experience.



Choosing SEO Instead Of Other Improvements



The curriculum required choosing one area:



Responsive design

SEO

Accessibility



AI could explain all three, but I decided to focus on SEO because it was the area where the site had the biggest gap between looking complete and actually behaving like a production website.



Reviewing And Accepting Changes



AI proposed code changes, but I was responsible for understanding them before committing.



Examples include:



The use Meta hook

Heading hierarchy changes

Semantic HTML updates

Open Graph implementation



The important decision was not writing the code. It was determining whether the code improved the project and matched the goal.



What I Learned



The biggest lesson from this project is that AI is extremely effective at implementation speed, but it is not responsible for product decisions.



AI generated code faster than I could write it myself.



However, I still had to decide:



What to build

What quality meant

Which improvements mattered most

Whether the generated solution was actually correct



The value came from combining AI's speed with human judgement rather than replacing one with the other.
